## Высокие (неверное поведение / уязвимость)

### 1. Path traversal в `apply_organization` — файлы уходят за пределы рабочей папки
**Файл:** `src/backend/memo/services/fs.py:90` (до фикса)

`folder_name` из тела запроса `/organize/apply` добавлялся к `abs_base` без
валидации: `dest_dir = os.path.join(abs_base, folder_name)`. Значение вида
`"../../Desktop"` после нормализации через `os.path.abspath` давало путь за пределами
рабочей директории. `shutil.move` выполнял перемещение успешно.

**Сценарий:** пользователь редактирует превью организации и переименовывает папку
кластера в `"../../Temp"` → все файлы кластера перемещаются в системную временную
директорию.

**Статус: ИСПРАВЛЕНО** — добавлена проверка `if "/" in folder_name or "\\" in folder_name or ".." in folder_name: raise ValueError`.
Тесты: `test_apply_organization_rejects_path_traversal`, `…_rejects_separator_in_folder_name`.

### 2. `suggest_filename` пропускает `..` → `save_document` отклоняет сгенерированное имя
**Файл:** `src/backend/memo/services/doc_generator.py:51` (до фикса)

`_INVALID_CHARS` не включает `.`, а `strip(". ")` удаляет только крайние точки.
Heading вида `# Протокол..2025` давал `suggest_filename` → `"Протокол..2025.md"`.
`save_document` проверяет `if ".." in filename` и бросает `ValueError`. Пользователь
видел 400-ошибку при попытке сохранить файл с авто-предложенным именем.

**Сценарий:** модель генерирует документ с заголовком `# Приложение..2` (что
реально встречается в деловых документах) → имя не проходит проверку в
`save_document`, хотя в UI выглядит корректно.

**Статус: ИСПРАВЛЕНО** — добавлено `re.sub(r'\.{2,}', '.', name)` до `strip`.
Тест: `test_suggest_filename_collapses_double_dots`.

## Средние

### 3. Файлы со статусом `error` вечно оказываются в «Разное»
**Файл:** `src/backend/memo/services/organizer.py:159` (до фикса)

Условие `row.status in ("stale",)` не включало `"error"`. Файлы с ошибкой
индексации (например, повреждённые PDF) не переиндексировались при запуске
авто-организации. Поскольку у них нет векторов в Chroma, они всегда попадали
в `misc_paths` → «Разное» без каких-либо пояснений для пользователя.

**Сценарий:** в папке есть файл `report.pdf`, который однажды дал ошибку при
индексации. Каждый запуск «Организовать» помещает его в «Разное», даже если
проблема с файлом была временной (например, файл был заблокирован).

**Статус: ИСПРАВЛЕНО** — условие изменено на `row.status != "indexed"`, что
охватывает `"stale"`, `"error"`, `"indexing"` и отсутствующие записи.

### 4. Частичный откат при `apply_organization` молча проглатывает ошибки
**Файл:** `src/backend/memo/services/fs.py:108`

При сбое в середине перемещения файлов откат выполняет `shutil.move(dest, src)`
для каждого уже перемещённого файла. Если одна из откатных операций сама бросает
исключение (`except Exception: pass`), файл остаётся в `dest`, а не возвращается
в `src`. Вызывающий получает `ValueError("Ошибка применения организации")` без
информации о том, какие файлы не удалось откатить.

**Сценарий:** 5 файлов перемещены успешно, 6-й — ошибка. Откат: файлы 5, 4, 3
возвращены успешно, файл 2 — нет (папка-источник была удалена другим процессом) →
файл остаётся в новой папке без предупреждения.

**Статус: не исправлено** — архитектурно ограничено (на уровне FS rollback-гарантии
полной атомарности в ОС нет без транзакционной FS). Задокументировано как
известное ограничение.

## Низкие (качество кода)

### 5. Дублирование `_INVALID_CHARS` в двух модулях
**Файлы:** `services/doc_generator.py:10` и `services/organizer.py:127`

Один и тот же паттерн `r'[\\/:*?"<>|\x00-\x1f]'` определён дважды. При изменении
правил именования файлов нужно будет обновить оба места.

**Рекомендация:** вынести в `memo/utils/filename.py` или в `memo/services/fs.py`
как экспортируемую константу.

### 6. Двойной запрос к Chroma в `analyze()`
**Файл:** `src/backend/memo/services/organizer.py:41` и `:190`

`_document_vectors` запрашивает Chroma с `include=["embeddings"]`, затем в `analyze`
идёт второй запрос с `include=["documents"]` для тех же файлов. Два round-trip
вместо одного (`include=["embeddings", "documents"]`) не влияет на корректность,
но увеличивает время анализа при большом числе файлов.

## Что проверено и НЕ является дефектом

- SSE-парсер `generate.ts`/`chat.ts`: `if (done) break` до `decoder.decode` — корректно. В Web Streams API при `done: true` поле `value` равно `undefined`; последние данные доставляются в предыдущем чтении с `done: false`.
- `suggest_filename` не обрабатывает `thinking`-события в `DocGenerator.tsx` — намеренно: в режиме генерации документа рассуждения модели скрыты (отображается только итоговый документ).
- `apply_organization` использует `exist_ok=True` при `os.makedirs` — корректно; папки создаются не дважды.
- `isApplying` в `OrganizePreview` не сбрасывается явно при успехе — корректно: `onDone` вызывает `setOrganizeMode(false)`, компонент размонтируется, локальное состояние уничтожается.
