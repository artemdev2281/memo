## UC-01: Индексация документов

**Актор:** Пользователь  
**Предусловие:** Приложение открыто, в левой панели отображаются файлы

### Статусы документов

| Иконка | Статус |
|---|---|
| 📄 | Не проиндексирован |
| 📄✅ | Проиндексирован |
| 📄⚠️ | Проиндексирован, но файл изменился (индекс устарел) |
| 📄⏳ | Индексация в процессе |
| 📄❌ | Ошибка индексации |

### Основной сценарий — первичная индексация

1. **Выбор объектов.** Пользователь выбирает файлы или папку в левой панели через чекбоксы. Для папки доступен toggle «Включая подпапки».
2. **Запуск.** Кнопка «Проиндексировать» в тулбаре левой панели (активна при наличии выбранных объектов).
3. **Процесс.** Файлы индексируются последовательно. Каждый обрабатываемый файл получает статус ⏳. В тулбаре отображается прогресс: «Индексация: 3/11».
4. **Результат.** Проиндексированные файлы получают статус ✅. В тулбаре краткое резюме: «Проиндексировано: 11 файлов».

### Сценарий — переиндексация устаревшего индекса

1. **Обнаружение изменений.** Система автоматически отслеживает изменения файлов. Изменённый файл получает статус ⚠️.
2. **Переиндексация.** Два способа:
   - вручную: выбрать файлы со статусом ⚠️ → «Проиндексировать»
   - автоматически: кнопка «Обновить устаревшие» — выбирает все файлы ⚠️ и запускает переиндексацию
3. **Результат.** Статус ⚠️ меняется на ✅.

### Альтернативные сценарии

- **Формат не поддерживается** — файл получает статус ❌, подсказка: «Формат не поддерживается».
- **Файл повреждён** — статус ❌, подсказка: «Не удалось прочитать файл».
- **Ollama недоступна** — процесс прерывается, сообщение об ошибке. Уже проиндексированные файлы сохраняют статус ✅.

### Ограничения

- Система не индексирует файлы автоматически без действия пользователя (за исключением Q&A без предварительной индексации)
- Индексация не изменяет содержимое файлов

## UC-02: Q&A по документам

**Актор:** Пользователь  
**Предусловие:** Приложение открыто

### Основной сценарий

1. **Выбор или создание чата.** В правой панели — список чатов. Пользователь выбирает существующий чат или создаёт новый кнопкой «Новый чат». Название генерируется автоматически по первому вопросу; пользователь может переименовать его в любой момент (инлайн-редактирование).

2. **Выбор контекста.** В левой панели пользователь определяет контекст одним из способов:
   - выбирает папку (toggle «Включая подпапки»)
   - отмечает отдельные файлы через чекбоксы
   - ничего не выбирает — AI отвечает без контекста документов  
   
   Текущий контекст отображается в правой панели (пример: «Контекст: договор.pdf, отчёт.docx»).

3. **Выбор модели.** Выпадающий список с моделями, установленными в Ollama. Выбор сохраняется между сессиями.

4. **Вопрос.** Пользователь вводит вопрос и отправляет.

5. **Обработка.** Индикатор загрузки. Система индексирует документы контекста (если ещё не проиндексированы) и выполняет RAG-поиск.

6. **Ответ.** AI выводит ответ. Если использовался контекст — под ответом список источников (названия файлов).

   > Пример:  
   > *Срок действия договора — 12 месяцев с момента подписания.*  
   > Источники: `договор_2024.pdf`

7. **Продолжение диалога.** Пользователь продолжает диалог. Контекст и модель сохраняются в рамках чата. История сохраняется между сессиями.

### Альтернативные сценарии

- **Ответ не найден** — AI сообщает: «В выбранных документах не найдено информации по этому вопросу».
- **Контекст не выбран** — AI отвечает на основе знаний модели, без ссылок на файлы.
- **Контекст пустой или форматы не поддерживаются** — предупреждение: «В выбранном контексте нет документов для анализа».
- **Файлы с устаревшим индексом в контексте** — предупреждение: «Некоторые документы изменились, индекс может быть устаревшим».
- **Ollama недоступна** — сообщение: «Убедитесь, что Ollama запущена».

### Ограничения

- Система не обращается к интернету
- Система не изменяет файлы

## UC-03: Авто-организация файлов

**Актор:** Пользователь  
**Предусловие:** Приложение открыто, в левой панели выбрана папка с документами

### Основной сценарий

1. **Запуск.** Два способа из левой панели:
   - правая кнопка мыши на папке → «Организовать файлы» в контекстном меню
   - отдельная кнопка в тулбаре левой панели (активна при выделенной папке)

2. **Настройка области.** В левой панели появляется inline-блок с опцией «Включая подпапки» (toggle) и кнопкой «Анализировать».

3. **Анализ.** В левой панели отображается прогресс: «Анализирую файлы... 4/11». Правая панель в это время доступна для других действий.

4. **Превью структуры.** AI показывает предложенное дерево «до/после» в левой панели. Структура плоская — один уровень вложенности. Файлы без чёткой категории помещаются в папку «Разное».

   ```
   До:                   После:
   📁 docs/              📁 docs/
     договор.pdf           📁 Договоры/
     отчёт_Q3.docx             договор.pdf
     план.md               📁 Отчёты/
     рецепт.txt                отчёт_Q3.docx
     заметка.txt           📁 Планирование/
                               план.md
                           📁 Разное/
                               рецепт.txt
                               заметка.txt
   ```

5. **Редактирование превью.** Пользователь работает с превью как с обычным файловым менеджером:
   - переименовать папку — инлайн-редактирование
   - переместить файл в другую папку — drag & drop
   - создать новую папку вручную
   - удалить папку (файлы перемещаются в «Разное»)
   
   Все изменения только в превью — файлы на диске не тронуты.

6. **Подтверждение.** Кнопка «Применить» в левой панели. Файлы физически перемещаются, папки создаются на диске. Левая панель обновляется.

7. **Результат.** В левой панели краткое резюме: «Создано папок: 4, перемещено файлов: 11». Через несколько секунд панель возвращается в обычный режим.

### Альтернативные сценарии

- **Отмена до применения** — кнопка «Отмена» на шагах 4–5, превью закрывается, файлы не перемещаются.
- **Папка пустая** — сообщение: «Нет файлов для анализа».
- **Все файлы одной темы** — AI создаёт одну папку, предупреждение: «Все документы относятся к одной теме — организация может не потребоваться».

### Ограничения

- Система не переименовывает сами файлы
- Система не удаляет файлы (только перемещает)
- Глубина структуры — один уровень (плоская организация)

## UC-04: Создание документа

**Актор:** Пользователь  
**Предусловие:** Приложение открыто, выбрана папка назначения

### Основной сценарий

1. **Запуск режима.** Пользователь нажимает кнопку «Создать документ» в правой панели. Панель переключается в режим создания.

2. **Выбор контекста (опционально).** Аналогично Q&A — пользователь может выбрать папку или файлы в левой панели. Если контекст выбран, AI опирается на него при генерации. Если не выбран — генерирует по запросу без контекста.

3. **Выбор формата.** Переключатель: **.txt** или **.md**.
   - `.txt` — plain text, без форматирования
   - `.md` — AI применяет Markdown-разметку (заголовки, списки, выделение)

4. **Запрос.** Пользователь описывает документ в свободной форме:
   > «Создай шаблон еженедельного отчёта»  
   > «Напиши заметку о встрече: обсуждали бюджет Q4, решили отложить до января»

5. **Генерация.** Индикатор загрузки. AI генерирует содержимое с учётом выбранного формата и контекста.

6. **Превью.** Сгенерированный текст отображается в правой панели. Для `.md` — рендеринг Markdown (итоговый вид файла).

7. **Сохранение.** Кнопка «Сохранить». Система предлагает автоматически сгенерированное имя файла — пользователь может изменить. Файл сохраняется в текущую папку.

8. **Результат.** Файл появляется в левой панели. Правая панель остаётся в режиме создания для следующего документа.

### Альтернативные сценарии

- **Запрос слишком неконкретный** — AI генерирует базовую структуру с пояснением: «Уточните запрос для более точного результата».
- **Папка не выбрана** — предупреждение: «Выберите папку для сохранения».

### Ограничения

- Система не редактирует существующие файлы
- Система создаёт только `.txt` и `.md` (не DOCX, не PDF)
- Файл не сохраняется автоматически без подтверждения пользователя
