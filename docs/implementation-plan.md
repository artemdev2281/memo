# План реализации memo

## Context

Проект **memo** — десктопное приложение для локальной работы с документами с встроенным AI-ассистентом (учебный проект, МТУСИ, БВТ2403). Документация в `docs/` полностью описывает спецификацию: технологический стек (Tauri 2 + React/TS + Python/FastAPI + ChromaDB + Ollama), 4 use case (индексация, Q&A, авто-организация, создание документов), модули backend, REST API, схемы БД.

**Цель плана:** разбить реализацию на вертикальные срезы по UC, чтобы после каждого этапа была работающая фича end-to-end. MVP = UC-01 + UC-02 (ядро ценности — RAG). UC-03 и UC-04 идут после MVP.

**Стратегические решения:**
- Поэтапно: skeleton → UC-01 → UC-02 (MVP) → UC-04 → UC-03 → polish.
- Базовое тестирование: pytest для ключевых модулей backend, ручное тестирование UI.

**Диагностика латентности think=OFF (среда, не код):** на Ollama 0.9.2 + qwen3:4b
модель игнорирует `think=false` и всё равно генерирует полный CoT (~25с скрытой
фазы) до `</think>`; `/no_think` не помогает. Ответ стримится корректно, но
первый токен — после рассуждений. Это особенность модели/сборки, не дефект кода.

**Код-ревью:** результаты в `docs/code-review-stage2-hardening.md` — 5 находок
(2 высокие, 3 средние). Открытые на момент завершения этапа: рассогласование
`(mtime,size)` с хэшем при правке в окне индексации (#1), ложное stale-предупреждение
из-за `error`-файлов (#2), блокирующее хэширование на event loop (#3), потеря
начала ответа при content-до-thinking (#4), обрыв SSE без события об ошибке (#5).

## Ключевые архитектурные решения

1. **Sidecar Python.** Порт выбирается динамически (`_free_port()`), передаётся через stdout (`MEMO_PORT=N`), читается Rust-оболочкой и передаётся frontend через Tauri command.

2. **SSE для стриминга.** Server-Sent Events для генерации LLM и прогресса индексации. На frontend — `fetch` + `ReadableStream`.

3. **State management.** Zustand для глобального состояния (контекст, модель, чат, статусы файлов).

4. **Переиспользование эмбеддингов.** Документный вектор для Organizer = среднее чанков из Chroma (не хранится отдельно).

5. **Тестовые фикстуры.** `tests/fixtures/` — минимальные `.pdf`, `.docx`, `.md`, `.txt`; Ollama мокируется, Chroma в `:memory:`.

## Что НЕ делаем в первой итерации

- Аутентификация, мульти-юзер — проект однопользовательский.
- Облачные LLM — принцип "полная локальность".
- Глубокая иерархия в Organizer — только плоская структура.
- Редактирование существующих документов через AI.
- Форматы `.xlsx`, `.pptx`, изображения.
- E2E-тесты через Playwright.
